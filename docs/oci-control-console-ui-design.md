# Oracle Cloud 控制台 UI 与功能层级设计文档

## 1. 产品定位

本项目定位为 **Oracle Cloud Infrastructure 专用的机器生命周期与自动化控制台**。

它不是多云平台，也不是通用服务器面板。当前阶段只围绕 Oracle Cloud / OCI 设计，所有页面、字段、任务、权限、资源模型都服务于 OCI 的实例创建、实例管理、扩容、降级、自动创建、任务追踪和审计。

核心目标：

- 管理 OCI API Key 与 Profile。
- 管理 OCI Compute Instance。
- 支持创建、启动、停止、重启、删除、保留启动盘删除。
- 支持实例升配、降配、Shape 修改。
- 支持模板化创建机器。
- 支持资源池与自动化规则。
- 支持任务中心、失败追踪、审计日志。
- 支持权限分级，降低误删、误扩容、误操作风险。

## 2. 产品边界

### 2.1 当前只做的内容

- Oracle Cloud / OCI API Key 管理。
- OCI Profile 管理。
- OCI Region / Compartment 管理。
- OCI Compute 实例管理。
- OCI Instance Configuration 模板能力。
- OCI Instance Pool / Autoscaling 的 UI 预留与逐步接入。
- OCI Work Request 任务追踪。
- OCI Monitoring / Audit 的 UI 对接预留。

### 2.2 当前明确不做的内容

- 不展示 AWS、Azure、Google Cloud、DigitalOcean、Cloudflare、SolusVM、VirtFusion 等入口。
- 不设计多云统一资源模型。
- 不设计跨云迁移、跨云同步、跨云账单。
- 不在前端保存 OCI 私钥。
- 不让前端直接调用 OCI API。
- 不把删除、扩容、重装等高风险操作做成无确认的快捷按钮。

## 4. 顶部全局栏设计

顶部全局栏承担“全局上下文”和“系统状态”的职责。

### 4.1 顶部字段

```text
R·Cloud·SSH / Oracle Cloud
Profile: DEFAULT
Region: ap-singapore-1
Compartment: production
CPU / MEM / NET / APP
同步
语言
主题
设置
退出
```

### 4.2 设计要求

- 移除其它云厂商标签栏。
- 当前 Profile、Region、Compartment 必须持续可见。
- 用户切换 Profile、Region、Compartment 时，实例列表、模板、资源池、任务中心都要刷新。
- 如果 Profile 未通过连接测试，顶部显示红色异常状态。
- 如果当前 Region 未启用或不可访问，顶部显示黄色警告状态。

### 4.3 推荐布局

```text
┌──────────────────────────────────────────────────────────────────────┐
│ R·Cloud·SSH  Oracle Cloud   Profile DEFAULT   Region ap-singapore-1  │
│ Compartment production       CPU 3% MEM 2.2/26 NET ↓14.8K ↑10.2K     │
│ [同步] [语言] [主题] [设置] [退出]                                    │
```

创建页关键原则：

- 默认推荐从模板创建。
- 空白创建面向高级用户。
- 批量创建必须显示预计创建数量和资源消耗。
- 创建前必须进入预检查。
- 提交后跳转任务中心或弹出任务进度抽屉。

- 删除资源池不等于删除实例，必须单独确认。

## 7. 关键操作流程

## 7.1 创建实例流程

```text
选择 Profile / Region / Compartment
-> 选择模板或空白创建
-> 填写实例配置
-> 填写网络与存储
-> 预检查
-> 提交创建任务
-> 后端调用 OCI API
-> 记录 OCI Request ID / Work Request ID
-> 轮询状态
-> 同步实例信息
-> 创建成功或失败
```

## 7.2 升级 / 降级流程

```text
选择实例
-> 点击升降级
-> 选择目标 Shape / OCPU / 内存
-> 检查镜像兼容性
-> 检查服务限额
-> 提醒实例可能重启
-> 用户确认
-> 创建任务
-> 调用 OCI UpdateInstance
-> 等待完成
-> 验证实例状态
-> 写入任务和审计日志
```

## 7.3 删除实例流程

```text
选择实例
-> 点击更多
-> 删除
-> 选择是否保留启动盘
-> 显示风险提示
-> 输入实例名称确认
-> 创建删除任务
-> 调用 OCI TerminateInstance
-> 等待完成
-> 同步状态
-> 写入审计日志
```

## 7.4 自动创建实例流程

```text
自动化规则触发
-> 检查规则状态
-> 检查资源池上限
-> 检查冷却时间
-> 检查 Profile 状态
-> 使用绑定模板生成创建参数
-> 创建任务
-> 调用 OCI API
-> 成功后加入资源池
-> 失败后记录原因并按规则重试
```

## 11. 当前界面改造建议

基于现有界面，建议做以下调整：

1. 移除云厂商切换栏，只保留 Oracle Cloud。
2. 顶部增加 Profile、Region、Compartment 三个全局选择器。
3. 左侧新增账号与密钥、模板管理、资源池、自动化规则、审计日志。
4. 实例卡片减少第一层按钮，只保留 SSH、关机、重启、升降级、更多。
5. 删除、重置镜像、删除保留启动盘放到更多菜单或危险操作区。
6. 创建实例改为七步向导。
7. 所有 OCI 操作提交后进入任务中心。
8. 所有失败任务保留 OCI Request ID、Work Request ID 和错误详情。
9. 自动化必须绑定模板和资源池。
10. 账号密钥必须后端加密存储，前端不可见私钥明文。
